# 그리드 레이아웃 시스템 제약사항

## Edge Cases & Constraints

### Input Domain

- **최소/최대 컬럼 수**: 화면 너비에 따라 자동 계산되지만, 최소 1개 컬럼은 보장되어야 함
- **아이템 크기 제한**:
  - 최소: 1x1 (150px × 150px)
  - 최대: 화면 너비를 초과할 수 있음 (오버플로우 에러 발생)
- **아이템 위치 유효성**:
  - 음수 위치는 허용되지 않음 (에러 발생)
  - 그리드 범위를 벗어난 위치는 오버플로우 에러 발생
  - 0 또는 음수 colSpan, rowSpan은 허용되지 않음 (에러 발생)

### State / Layout

- **아이템 겹침**: 겹침은 허용되지 않으며, 감지 시 에러 발생
- **동적 아이템 추가/제거**: 정적 배치이므로 런타임 변경은 고려하지 않음 (개발 시점 고정)
- **아이템 순서**: 수동 배치이므로 DOM 순서와 시각적 순서가 다를 수 있음
- **오버플로우 처리**: 그리드 영역을 벗어나면 에러 발생 (스크롤, 숨김, 재배치 없음)

### Performance

- **대량 아이템**: 100개 이상의 아이템이 있을 때 렌더링 성능 고려
- **리사이즈 이벤트**: 화면 크기 변경 시 그리드 재계산 최적화 필요

### UX / Accessibility

- **스크롤 동작**: 오버플로우 시 에러 발생하므로 스크롤은 고려하지 않음
- **터치 제스처**: 사용자 편집 기능 없음 (개발 시점 고정 배치)

## 검증 규칙 요약

### 필수 검증 항목

1. **위치 유효성 검증**

   - `col > 0`, `row > 0`
   - `colSpan > 0`, `rowSpan > 0`

2. **오버플로우 검증**

   - `col + colSpan <= 현재 컬럼 수`
   - 화면 크기 변경 시 재검증 필요

3. **겹침 검증**
   - 모든 아이템 쌍에 대해 겹침 여부 확인
   - 겹침 감지 시 에러 발생
