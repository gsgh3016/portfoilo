# 그리드 구조 규칙 테스트 시나리오

## R1. 그리드 컬럼 너비

### TS-R1.1: 컬럼 너비 고정값 검증

**ID**: TS-R1.1 | **규칙**: R1.1  
**Given**: 그리드 컨테이너가 생성됨  
**When**: 컬럼 너비를 설정할 때  
**Then**: 모든 컬럼의 너비는 150px로 고정됨

---

### TS-R1.2: 모든 환경에서 컬럼 너비 동일성

**ID**: TS-R1.2 | **규칙**: R1.2  
**Given**: 다양한 화면 크기 환경 (375px, 768px, 1920px)  
**When**: 그리드가 렌더링될 때  
**Then**: 모든 환경에서 컬럼 너비는 150px로 동일함

---

### TS-R1.3: 화면 너비에 따른 컬럼 수 자동 조정

**ID**: TS-R1.3 | **규칙**: R1.3  
**Given**: 화면 너비가 1920px일 때  
**When**: 그리드가 렌더링될 때  
**Then**: 컬럼 수는 12개

**Given**: 화면 너비가 768px일 때  
**When**: 그리드가 렌더링될 때  
**Then**: 컬럼 수는 5개

**Given**: 화면 너비가 375px일 때  
**When**: 그리드가 렌더링될 때  
**Then**: 컬럼 수는 2개

---

### TS-R1.4: 최소 컬럼 수 보장

**ID**: TS-R1.4 | **규칙**: R1.3 (제약사항 반영)  
**Given**: 매우 작은 화면 너비 (100px 이하)  
**When**: 그리드가 렌더링될 때  
**Then**: 최소 1개 컬럼은 보장되어야 함

---

## R2. 그리드 행 높이

### TS-R2.1: 행 높이 고정값 검증

**ID**: TS-R2.1 | **규칙**: R2.1  
**Given**: 그리드 컨테이너가 생성됨  
**When**: 행 높이를 설정할 때  
**Then**: 모든 행의 높이는 150px로 고정됨

---

### TS-R2.2: 행 높이와 컬럼 너비 동일성

**ID**: TS-R2.2 | **규칙**: R2.2  
**Given**: 그리드 컨테이너가 생성됨  
**When**: 행 높이와 컬럼 너비를 확인할 때  
**Then**: 행 높이(150px)는 컬럼 너비(150px)와 동일함

---

## R5. 그리드 간격(Gap)

### TS-R5.1: 그리드 간격 고정값 검증

**ID**: TS-R5.1 | **규칙**: R5.1  
**Given**: 그리드 컨테이너가 생성됨  
**When**: 간격을 설정할 때  
**Then**: 그리드 아이템 간 간격은 10px로 고정됨

---

### TS-R5.2: 간격 설정 가능성

**ID**: TS-R5.2 | **규칙**: R5.2  
**Given**: 그리드 컨테이너가 생성됨  
**When**: 간격을 명시하지 않을 때  
**Then**: 기본 간격은 10px임

**Given**: 그리드 컨테이너가 생성됨  
**When**: 간격을 20px로 설정할 때  
**Then**: 간격은 20px로 적용됨

---
